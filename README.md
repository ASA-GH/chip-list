# Adaptive Chip List Component with Overflow Popup

Тестовое задание для компании Сириус

React-компонент для отображения списка чипсов с автоматическим скрытием не влезающих элементов в попап.

## Технологии

- **React 19**
- **TypeScript**
- **LESS** (с использованием CSS Modules)
- **Webpack 5**
- **Jest** + React Testing Library

## Установка и запуск

**Требования:**

- Node.js `>=24.14.0`
- npm `^11.9.0` (указан в `packageManager`)

```bash
# Установка зависимостей
npm install

# Запуск в режиме разработки
npm start

# Сборка (автоматически запускает тесты перед билдом)
npm run build

# Запуск тестов
npm test
```

## Доступные скрипты

- **npm start** – запуск dev-сервера (Webpack Dev Server)
- **npm run** build – продакшн-сборка с предварительным прогоном тестов
- **npm test** – запуск тестов
- **npm run lint** – проверка кода с помощью ESLint
- **npm run format** – форматирование кода Prettier

## Описание компонентов

- **ChipList** – основной компонент. Принимает массив `chips`, отображает их в виде строки чипсов с использованием
  `ChipSlot`. С помощью хука `useChipOverflow` автоматически вычисляет, сколько элементов помещается в контейнер. Не
  поместившиеся чипсы скрываются, а вместо них появляется кнопка `...` (троеточие). При клике на неё открывается `Popup`,
  в котором отображаются все скрытые чипсы (также через `ChipSlot`). Поддерживает выбор чипсов через пропс `selectedIds` и
  колбэк `onChipSelect`. Стилизация через CSS-модули, можно добавить внешний класс.

- **Chip** – базовый элемент чипса. Принимает `content`, который может быть строкой или произвольным React-узлом (
  например, текст с иконкой). Поддерживает состояния выбора `selected` и блокировки `disabled`. При клике (если не
  `disabled`) вызывает колбэк `onClick`. Стилизация через CSS-модули с возможностью добавить внешний класс.

- **ChipSlot** – компонент-обёртка, который рендерит отдельный чипс в виде элемента списка `<li>`. Используется внутри
  `ChipList` для единообразного отображения чипсов как в основном списке, так и в `Popup`. Принимает пропсы для передачи в
  `Chip` (`content`, `selected`, `onClick`), а также технический проп marker для установки `data-атрибута` (используется в хуке
  `useChipOverflow` для поиска элементов). Благодаря `forwardRef` позволяет получить ссылку на DOM-элемент `<li>` (например, для
  измерения ширины или позиционирования `Popup`).

- **Popup** – `Popup` – всплывающее окно, которое рендерится в портал `ReactDOM.createPortal` в `document.body`. Появляется
  при `isOpen = true` и позиционируется относительно элемента `anchorRef` (кнопки `...`) с отступом 6px справа и снизу (с
  учётом transform: translateX(-100%), чтобы окно смещалось влево от якоря). Закрывается при нажатии клавиши `Escape` (
  через хук `useKeyDown`) или при клике вне области `Popup` и вне элемента `anchorRef` (через хук `useClickOutside`).
  Координаты пересчитываются в `useLayoutEffect` при открытии, чтобы учесть возможное изменение позиции кнопки. Содержимое
  передаётся через `children`. В проекте используется для отображения списка скрытых чипсов.

- **App** – демонстрационный компонент, который подготавливает тестовые данные `generateChips` и управляет состоянием
  выбранных чипсов. Показывает пример интеграции `ChipList`.

## API компонентов

### ChipList

- **`chips: ChipsList | []`** — массив `ChipData` для отображения в виде чипсов.
- **`selectedIds?: string[]`** — массив выбранных идентификаторов (определяет активные чипсы).
- **`onChipSelect?: (id: string) => void`** — колбэк, вызываемый при клике на чипс. Получает `id` выбранного элемента.
- **`className?: string`** — дополнительный CSS-класс для корневого элемента.

### Chip

- **`content?: string | ReactNode`** — содержимое чипса (текст или JSX-элемент, например, с иконкой).
- **`selected?: boolean`** — визуально выделяет чипс как выбранный (по умолчанию `false`).
- **`disabled?: boolean`** — блокирует взаимодействие и применяет соответствующий стиль (по умолчанию `false`).
- **`onClick?: () => void`** — вызывается при клике на чипс (игнорируется, если `disabled`).
- **`className?: string`** — дополнительный CSS-класс для кастомизации.

### ChipSlot

- **`id?: string`** — уникальный идентификатор чипса (передаётся в `onClick`).
- **`content: string | ReactNode`** — содержимое чипса.
- **`selected?: boolean`** — флаг выбранного состояния (передаётся в `Chip`).
- **`onClick: (id?: string) => void`** — колбэк клика; вызывается с `id` чипса (если передан).
- **`className?: string`** — CSS-класс для элемента `<li>`.
- **`marker?: string`** — имя data-атрибута, который будет установлен на `<li>` со значением `true` (например,
  `data-chip-slot`).

### Popup

- **`isOpen: boolean`** — управляет видимостью `Popup`.
- **`onClose: () => void`** — вызывается при запросе закрытия (клик вне `Popup`, нажатие `Escape`).
- **`anchorRef?: React.RefObject<HTMLElement>`** — ссылка на элемент (кнопку), относительно которого позиционируется
  попап.
- **`children: ReactNode`** — содержимое, отображаемое внутри `Popup`.
- **`className?: string`** — дополнительный CSS-класс для корневого элемента `Popup`.

## Стилизация и переменные

В проекте используется **LESS** с **CSS Modules** (`*.module.less`) для изоляции стилей компонентов. Все глобальные
переменные (цвета, типографика, отступы, эффекты и z-index) вынесены в отдельный файл `src/styles/variables.less` и
организованы по принципу **"сырые значения → семантические роли"**.

### Пример организации

```less
// 1. Сырые цвета (Raw)
@raw-clr-white: #ffffff;
@raw-clr-gray-100: #eeeeee;
@raw-clr-blue-500: #007bff;

// Семантические роли (используются в компонентах)
@chip-bg: @raw-clr-gray-100;
@chip-bg-hover: @raw-clr-gray-200;
@chip-bg-sel: @raw-clr-blue-500;
```

- **Цвета** — сначала базовые значения, затем семантические алиасы для конкретных элементов (`chip-bg`, `popup-shadow`).
- **Типографика** — базовые размеры (`@raw-fs-12`) и семантические (`@fs-small`).
- **Отступы и размеры** — сырые единицы (`@raw-sp-8`) превращаются в логические переменные (`@chip-pad`,
  `@container-max-w`).
- **Эффекты** — радиусы скругления, тени, transition.
- **Z-index** — управление слоями через семантические переменные (`@popup-z-index`).

### Такой подход обеспечивает:

- Единообразие стилей во всём проекте.
- Лёгкую смену темы (достаточно изменить семантические переменные).
- Читаемость и поддерживаемость кода.
